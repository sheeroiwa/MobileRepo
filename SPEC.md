# Техническое задание. Финансовый трекер: Учёт доходов и расходов

## 1. Общие сведения

| Параметр | Значение |
|----------|----------|
| **Название проекта** | FinTrackApp (Финансовый трекер) |
| **Цель** | Помочь пользователю вести учёт доходов и расходов, анализировать бюджет, экспортировать данные |
| **Целевая аудитория** | Студенты, люди, желающие контролировать личный бюджет без интернета |
| **Платформа** | Android |
| **Минимальная версия Android** | API 24 (Android 7.0 Nougat) |
| **Целевая версия Android** | API 34 (Android 14) |
| **Язык разработки** | Java 11 |
| **Режим работы** | Полностью офлайн, без интернета и облачных сервисов |

## 2. Функциональные требования (Must Have / Should Have / Could Have)

### Must Have (обязательно к реализации)
- [ ] Добавление транзакции (доход/расход)
- [ ] Редактирование транзакции
- [ ] Удаление транзакции
- [ ] Просмотр списка всех транзакций
- [ ] Фильтрация по типу (доходы / расходы)
- [ ] Сортировка по дате (сначала новые/старые)
- [ ] Просмотр текущего баланса (доходы — расходы)

### Should Have (важно, но можно позже)
- [ ] Фильтр по категориям
- [ ] Сортировка по сумме
- [ ] Круговая диаграмма расходов по категориям
- [ ] Экспорт данных в CSV

### Could Have (желательно, но не критично)
- [ ] Тёмная тема
- [ ] Импорт из CSV
- [ ] Кастомные иконки для категорий
- [ ] График динамики доходов/расходов по дням

### Won't Have (сознательно не делаем)
- Синхронизация с облаком
- Онлайн-курсы валют
- Push-уведомления
- Многопользовательский режим
- Платные API или библиотеки

## 3. Экраны и пользовательские сценарии

### Экран 1: Список транзакций (TransactionListFragment)
**Сценарий: Просмотр и фильтрация**

| Пункт | Описание |
|-------|----------|
| **Актор** | Пользователь |
| **Предусловие** | В БД есть хотя бы одна транзакция |
| **Основной поток** | 1. Открыть приложение → виден список транзакций<br>2. Сверху отображается текущий баланс<br>3. Кнопка «+» для добавления<br>4. Кнопки фильтра «Все» / «Доходы» / «Расходы»<br>5. Каждый элемент списка: сумма, категория, дата |
| **Постусловие** | Список обновляется при изменении фильтра |

### Экран 2: Добавление / редактирование (AddEditTransactionFragment)
**Сценарий: Добавление новой транзакции**

| Пункт | Описание |
|-------|----------|
| **Актор** | Пользователь |
| **Предусловие** | Приложение открыто |
| **Основной поток** | 1. Нажать «+» → открывается форма<br>2. Ввести сумму (обязательно, число >0)<br>3. Выбрать тип: доход / расход<br>4. Выбрать категорию из списка<br>5. Выбрать дату (DatePicker)<br>6. Ввести описание (опционально)<br>7. Нажать «Сохранить» |
| **Альтернативные потоки** | • Сумма пустая или ≤0 → ошибка «Введите сумму»<br>• Нажатие «Назад» → спросить «Отменить изменения?»<br>• В режиме редактирования — загрузить существующие данные |
| **Постусловие** | Транзакция сохранена в БД, список обновлён |

### Экран 3: Статистика (StatisticsFragment)
**Сценарий: Просмотр диаграммы расходов**

| Пункт | Описание |
|-------|----------|
| **Актор** | Пользователь |
| **Предусловие** | Есть хотя бы одна транзакция расхода |
| **Основной поток** | 1. Перейти на вкладку «Статистика»<br>2. Увидеть круговую диаграмму расходов по категориям<br>3. Легенда: название категории + сумма + процент |
| **Постусловие** | Диаграмма обновляется при изменении данных |

### Экран 4: Экспорт (ExportFragment или кнопка)
**Сценарий: Экспорт в CSV**

| Пункт | Описание |
|-------|----------|
| **Актор** | Пользователь |
| **Предусловие** | В БД есть хотя бы одна транзакция |
| **Основной поток** | 1. Нажать «Экспорт» → открыть Storage Access Framework<br>2. Выбрать папку и имя файла<br>3. Система генерирует файл `transactions_YYYYMMDD.csv`<br>4. Показать Toast: «Экспорт завершён» |
| **Альтернатива** | Пользователь отменил выбор → ничего не происходит |
| **Постусловие** | CSV-файл сохранён в выбранную папку |

## 4. Нефункциональные требования

| Категория | Требование |
|-----------|-------------|
| **Производительность** | Список из 500 транзакций прокручивается без фризов |
| **Память** | Потребление RAM ≤ 150 МБ, нет утечек при повороте |
| **Надёжность** | Корректная обработка пустой БД, некорректного ввода |
| **Совместимость** | API 24–34, экраны 4.7–6.7 дюймов |
| **Безопасность** | Данные локально, доступ к файлам только через SAF |
| **Офлайн** | 100% офлайн, без интернета |

## 5. Краевые случаи и обработка ошибок

| Ситуация | Поведение |
|----------|-----------|
| Пустой список транзакций | Показать «Нет транзакций» + иконку |
| Ошибка при сохранении | Диалог «Ошибка, попробуйте снова» |
| Нет места на диске при экспорте | Toast «Недостаточно места» |
| Поворот экрана | Состояние сохраняется через ViewModel |
| Приложение убито в фоне | Данные восстанавливаются из Room |
